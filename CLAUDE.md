# wade

TUI file browser in .NET 10. Runs on Windows Terminal and Unix/WSL terminals.

## Build & Test

```bash
dotnet build Wade.slnx
dotnet test Wade.slnx
dotnet run --project src/Wade [path]
```

## Key Conventions

- **No TUI frameworks** — rendering uses raw VT/ANSI escape sequences only (no Terminal.Gui, no Spectre.Console)
- **NativeAOT-ready** — avoid reflection-heavy patterns; `[GeneratedRegex]` is AOT-safe and used in `MarkdownLanguage`
- **Platform separator** — use `Path.DirectorySeparatorChar`, not hardcoded `/` or `\`
- **Windows console input** — do NOT enable `ENABLE_VIRTUAL_TERMINAL_INPUT`; input uses Win32 `ReadConsoleInput` (not `Console.ReadKey`). VT processing is output-side only. Mouse input uses `ENABLE_MOUSE_INPUT` (Windows) and SGR mouse mode VT sequences (Unix).
- **Unix console input** — raw mode via `cfmakeraw` P/Invoke on `/dev/tty`; input uses libc `read()`/`poll()` syscalls (not `Console.ReadKey` or `FileStream`). Arrow keys arrive as SS3 sequences (`ESC O A/B/C/D`) in alternate screen mode, not CSI.
- **Sixel image preview** — uses SixLabors.ImageSharp for decoding/scaling and a custom median-cut Sixel encoder (`Wade.Imaging`). Sixel data bypasses `ScreenBuffer` (written directly to stdout after flush). Gated by `ImagePreviewsEnabled` config flag (default: on) AND terminal Sixel capability detection (Windows Terminal via `WT_SESSION`, Unix via DA1 query). Only written in allowlisted modes (`Normal`, `Search`, `ExpandedPreview`) — all modal overlays suppress it automatically. Cell pixel dimensions detected at startup for proper image scaling (defaults: 8×16 px/cell). Non-image formats (PDF) use a convert-to-image pipeline (`IImageConverter` / `ImageConverter`) that shells out to external CLI tools (e.g. `pdftopng`) and feeds the result through `ImagePreview.Load()`.
- **Git integration** — git status display via `git status --porcelain=v1` (NativeAOT-safe, no LibGit2Sharp). Branch name read from `.git/HEAD` (supports worktrees). Async via `GitStatusLoader` following the `DirectorySizeLoader` pattern. File/directory coloring by status (modified=yellow, staged=cyan, untracked=green, conflict=red). Nerd Font status icons rendered in a dedicated 2-char `StatusColWidth` column between the filename and size column (not inline with the file-type icon). Branch name in status bar is prefixed with a powerline branch icon (`U+E0A0`). Directory-level aggregate status (OR of child statuses). Gated by `git_status_enabled` config (default: on). Status map is a separate `Dictionary<string, GitFileStatus>` passed to `PaneRenderer`, not stored on `FileSystemEntry`. Diff preview available via the preview provider system (select "Git diff" from the action palette preview entries); shows colored `git diff` output in the preview pane for modified/staged files; `DiffLanguage` provides per-line `CharStyles` coloring (green=added, red=removed, cyan=hunk headers, dim gray=metadata). Git actions are prefixed with "Git:" in the action palette and only shown when actionable. Stage/unstage via `GitActionRunner` (async, follows `GitStatusLoader` pattern); `GitUtils.Stage`/`Unstage`/`StageAll`/`UnstageAll`/`Commit`/`Push`/`PushForceWithLease`/`Pull`/`PullRebase`/`Fetch` execute the corresponding git commands; network operations use 30s timeout. `GetAheadBehind` queries `git rev-list --count --left-right @{upstream}...HEAD`; result displayed in status bar as `↑N ↓M` after branch name; multi-select supported via `_markedPaths`; status auto-refreshes after each action via `GitActionCompleteEvent`. Properties overlay (`i` key) shows a "Git status" row with colored text matching PaneRenderer colors; `PropertiesOverlay.Render` takes an optional `GitFileStatus?` parameter; `FormatGitStatus` is `internal static` for testability.
- **File finder** — `Ctrl+F` recursive file search via `ScanFilesForFinder` (internal static, testable). Uses manual stack-based directory walk (not `RecurseSubdirectories`) so directories can be skipped before descending. `.git` directories are always excluded (even with `showHidden=true`). Dot-prefixed directories and hidden/system directories are skipped based on visibility settings. Depth limit 8, entry cap 10,000.
- **Cloud file placeholders** — OneDrive/Dropbox/cloud-synced files that aren't fully downloaded locally are detected via Windows Cloud Files API attributes (`FILE_ATTRIBUTE_RECALL_ON_DATA_ACCESS` 0x00400000, `FILE_ATTRIBUTE_RECALL_ON_OPEN` 0x00004000). Cast `FileInfo.Attributes` to `int` and check bits — no P/Invoke needed. Cloud placeholders keep their original file-type icon in the first column; a cloud icon (nf-md-cloud_outline U+F0163) is shown in the status column (same column as git status icons). Dim gray styling (140, 140, 160). Properties dialog shows the extension-based type label (e.g. "PDF", "C#") or "File"/"Directory" and adds "Cloud file" to the Attributes field. Metadata is shown via `FileMetadataProvider` (filename, "Cloud: not downloaded") — `MetadataProviderRegistry` returns only `FileMetadataProvider` for cloud placeholders. Visual preview is blocked at the registry level (`PreviewProviderRegistry` returns empty for `IsCloudPlaceholder`); the preview pane renders "[cloud file – not downloaded]" instead of attempting to read the file. Detection in `DirectoryContents.CheckIsCloudPlaceholder()`, stored on `FileSystemEntry.IsCloudPlaceholder`. Windows-only (always `false` on Unix).
- **Drive type detection** — `DriveTypeDetector.Detect(DriveInfo)` returns `DriveMediaType` enum (`Unknown`, `Ssd`, `Hdd`, `Network`, `Removable`). Windows SSD/HDD detection via `DeviceIoControl` seek penalty query (`IOCTL_STORAGE_QUERY_PROPERTY` with `StorageDeviceSeekPenaltyProperty`). Linux via `/sys/block/<device>/queue/rotational`. Falls back to `Unknown` on failure. Stored on `FileSystemEntry.DriveMediaType` (default `Unknown`). Populated in `DirectoryContents.GetDriveEntries()`. Properties overlay shows "SSD"/"HDD" instead of "Fixed" for detected drives. P/Invoke declarations are private inside `DriveTypeDetector`. Parsing logic extracted into `internal static` methods for testability (`ParseRotationalValue`, `ParseSeekPenaltyResult`, `ExtractBaseDevice`).
- **Inline directory sizes** — `InlineDirSizeLoader` computes directory sizes asynchronously for all subdirectories in the current listing. Streaming results: injects one `InlineDirSizeReadyEvent` per directory as each completes. App.cs accumulates into `_inlineDirSizes` dictionary, passed to `PaneRenderer.RenderFileList` via `dirSizes` parameter. Gated per drive type via `App.ShouldComputeInlineDirSizes(DriveMediaType, WadeConfig)`: SSD on by default, HDD/Network off, Unknown disabled. Triggered from `RefreshGitStatus()` on every navigation. Config: `dir_size_ssd_enabled`, `dir_size_hdd_enabled`, `dir_size_network_enabled`. Config dialog items nested under "Show Size Column" with `EnabledWhen = () => SizeColumn`.
- **Preview provider system** — Two separate interfaces in `Wade.Preview`: `IPreviewProvider` for visual previews (text/image) and `IMetadataProvider` for structured metadata extraction. `IPreviewProvider` declares `CanPreview()` and returns a `PreviewResult` carrying text (`StyledLine[]`), image (Sixel data), or both. `IMetadataProvider` declares `CanProvideMetadata()` and returns `MetadataResult` with structured `MetadataSection[]`/`MetadataEntry[]`. `PreviewContext` carries pane dimensions, config flags (`ZipPreviewEnabled`, `ImagePreviewsEnabled`, `SixelSupported`, `ArchiveMetadataEnabled`), git status, `IsCloudPlaceholder`, and `IsBrokenSymlink`. `MetadataProviderRegistry` short-circuits for `IsBrokenSymlink`; for cloud placeholders, only `FileMetadataProvider` is returned (no file I/O — shows filename and "Cloud: not downloaded"); skips `ArchiveMetadataProvider` when `context.ArchiveMetadataEnabled` is false. `PreviewProviderRegistry` short-circuits for both `IsBrokenSymlink` and `IsCloudPlaceholder`. When `FilePreviewsEnabled` is false, App.cs returns an empty provider list instead of querying the registry. Preview providers in `PreviewProviders.cs` (registry order): `ImagePreviewProvider`, `PdfPreviewProvider`, `GlowMarkdownPreviewProvider`, `ZipContentsPreviewProvider`, `MsiPreviewProvider` (`.msi` — file table listing from MSI database; Windows-only), `TextPreviewProvider` (non-binary only via `FilePreview.IsBinary`), `DiffPreviewProvider`, `NonePreviewProvider` (always applicable, returns placeholder), `HexPreviewProvider` (always applicable, explicit opt-in). Binary files default to None (not Hex) because None precedes Hex in registry order. Metadata providers (registry order): `FileMetadataProvider` (all files and directories — filename as section header, cloud status, git status if applicable; always first), `ImageMetadataProvider` (`.png`/`.jpg`/`.jpeg`/`.gif`/`.bmp`/`.webp`/`.tga`/`.tiff`/`.pbm` — resolution, format, color depth, frame count via `Image.Identify()`; EXIF section with camera, date taken, exposure, focal length, GPS, software when present), `ExecutableMetadataProvider` (`.exe`/`.dll` — PE headers via `PEReader`, .NET assembly metadata via `MetadataReader`, `FileVersionInfo` on Windows; NativeAOT-safe), `OfficeMetadataProvider` (`.docx`/`.xlsx`/`.pptx`/`.dotx`/`.xltx`/`.potx` — OPC metadata from `docProps/core.xml` and `docProps/app.xml`), `MediaMetadataProvider` (audio/video — shells out to `ffprobe`/`mediainfo`; `ParseFfprobeJson`/`ParseMediainfoJson` are `internal static` for testability), `NuGetMetadataProvider` (`.nupkg`/`.snupkg` — `.nuspec` XML via `System.Xml.Linq` with local-name matching), `MsiMetadataProvider` (`.msi` — ProductName, Version, Manufacturer, ProductCode, UpgradeCode via Windows Installer database API; Windows-only), `ArchiveMetadataProvider` (all zip-based archives — file count, total/compressed size, compression ratio via `System.IO.Compression.ZipFile`; applies to all `ZipPreview.IsZipFile()` extensions; gated by `ArchiveMetadataEnabled` config), `PdfMetadataProvider` (`.pdf` — shells out to `pdfinfo`). `MetadataRenderer` (`Wade.UI`) converts structured `MetadataSection[]` to `StyledLine[]` with colored label/value styling. `MetadataProviderRegistry.GetApplicableProviders()` returns a `List<IMetadataProvider>` of **all** applicable providers for a file (not just the first match); their sections are combined in order before injecting a single `MetadataReadyEvent`. `PreviewLoader.BeginLoad(path, metadataProviders, previewProvider, context)` loads metadata from all providers first (iterating in list order, merging sections, injecting `MetadataReadyEvent`), then loads the preview. App.cs stores `_applicableMetadataProviders` (list, no active index — all sections are shown together). App.cs renders three display modes: metadata-only, preview-only, or metadata header above preview. `PropertiesOverlay.Render` accepts optional `MetadataSection[]` to show extracted metadata below file system properties. `PreviewProviderRegistry` provides an ordered list of applicable preview providers per file. Action palette supports submenus via a stack-based `ActionMenuLevel` model (`Wade.UI`). Preview providers are grouped under a "Change preview" submenu when multiple providers are available; also accessible directly via `p` keybinding. Combined text+image previews are supported via `CombinedPreviewReadyEvent`. To add a new metadata format: implement `IMetadataProvider` and add to `MetadataProviderRegistry.s_providers`. To add a new visual preview: implement `IPreviewProvider` and add to `PreviewProviderRegistry.s_providers`.
- **Filesystem auto-refresh** — `FileSystemWatcherManager` watches the current directory for changes (create, delete, rename, modify) and injects `FileSystemChangedEvent` into the `InputPipeline`. Follows the async loader pattern (`DirectorySizeLoader`, `GitStatusLoader`). 300ms debounce timer coalesces rapid events. Watcher auto-switches when navigating to a new directory (checked at the top of each render loop via `Watch()` which no-ops if path unchanged). `HandleFileSystemChanged` in App.cs preserves selection by name, invalidates the directory cache, refreshes git status, and forces a full redraw. Buffer overflow (`Error` event) triggers a full cache invalidation. NativeAOT-compatible.
- **Config dialog** — data-driven model in `Wade.UI`: `ConfigItem` (pure data: label, indent, format/toggle/cycle delegates, enabled predicate) and `ConfigDialogState` (owns mutable dialog state, builds items, handles navigation/toggle/cycle, copies to/from `WadeConfig` via `FromConfig`/`ApplyTo`). App.cs creates `ConfigDialogState` on dialog open, delegates input to its methods (`MoveUp`/`MoveDown`/`ToggleSelected`/`CycleNextSelected`/`CyclePrevSelected`), and renders from `Items`/`SelectedIndex`. No hardcoded index math — items are built once with platform-conditional insertion (Windows-only "Show System Files"). Hierarchy: "Show Right Pane" gates "Show File Details" (with archive/PDF/media detail sub-items gated by `FileMetadata`) and "Show File Previews" (with image/PDF/archive/markdown preview sub-items gated by `FilePreviews`). Disabled items are skipped during navigation. `ConfigDialogStateTests` covers state, navigation, toggles, enabled predicates, and round-tripping.
- **Avoid Ctrl+Shift+&lt;key&gt; keybindings** — Windows Terminal reserves many `Ctrl+Shift+` combinations (`C`, `V`, `P`, `W`, `Tab`, etc.) and `Ctrl+Tab`. Do not assign these as wade keybindings; they will be intercepted by the terminal and never reach the app.
- Uses `UseArtifactsOutput` — build output goes to `artifacts/`, not `bin/`/`obj/`
