# wade

TUI file browser in .NET 10. Runs on Windows Terminal and Unix/WSL terminals.

## Build & Test

```bash
dotnet build Wade.slnx
dotnet test Wade.slnx
dotnet run --project src/Wade [path]
```

## Key Conventions

- **No TUI frameworks** ŌĆö rendering uses raw VT/ANSI escape sequences only (no Terminal.Gui, no Spectre.Console)
- **NativeAOT-ready** ŌĆö avoid reflection-heavy patterns; `[GeneratedRegex]` is AOT-safe and used in `MarkdownLanguage`
- **Platform separator** ŌĆö use `Path.DirectorySeparatorChar`, not hardcoded `/` or `\`
- **Windows console input** ŌĆö do NOT enable `ENABLE_VIRTUAL_TERMINAL_INPUT`; input uses Win32 `ReadConsoleInput` (not `Console.ReadKey`). VT processing is output-side only. Mouse input uses `ENABLE_MOUSE_INPUT` (Windows) and SGR mouse mode VT sequences (Unix).
- **Unix console input** ŌĆö raw mode via `cfmakeraw` P/Invoke on `/dev/tty`; input uses libc `read()`/`poll()` syscalls (not `Console.ReadKey` or `FileStream`). Arrow keys arrive as SS3 sequences (`ESC O A/B/C/D`) in alternate screen mode, not CSI.
- **Sixel image preview** ŌĆö uses SixLabors.ImageSharp for decoding/scaling and a custom median-cut Sixel encoder (`Wade.Imaging`). Sixel data bypasses `ScreenBuffer` (written directly to stdout after flush). Gated by `ImagePreviewsEnabled` config flag (default: on) AND terminal Sixel capability detection (Windows Terminal via `WT_SESSION`, Unix via DA1 query). Only written in allowlisted modes (`Normal`, `Search`, `ExpandedPreview`) ŌĆö all modal overlays suppress it automatically. Cell pixel dimensions detected at startup for proper image scaling (defaults: 8├Ś16 px/cell). Non-image formats (PDF) use a convert-to-image pipeline (`IImageConverter` / `ImageConverter`) that shells out to external CLI tools (e.g. `pdftopng`) and feeds the result through `ImagePreview.Load()`.
- **Git integration** ŌĆö git status display via `git status --porcelain=v1` (NativeAOT-safe, no LibGit2Sharp). Branch name read from `.git/HEAD` (supports worktrees). Async via `GitStatusLoader` following the `DirectorySizeLoader` pattern. File/directory coloring by status (modified=yellow, staged=cyan, untracked=green, conflict=red). Nerd Font status icons rendered in a dedicated 2-char `StatusColWidth` column between the filename and size column (not inline with the file-type icon). Branch name in status bar is prefixed with a powerline branch icon (`U+E0A0`). Directory-level aggregate status (OR of child statuses). Gated by `git_status_enabled` config (default: on). Status map is a separate `Dictionary<string, GitFileStatus>` passed to `PaneRenderer`, not stored on `FileSystemEntry`. Diff preview available via the preview provider system (select "Git diff" from the action palette preview entries); shows colored `git diff` output in the preview pane for modified/staged files; `DiffLanguage` provides per-line `CharStyles` coloring (green=added, red=removed, cyan=hunk headers, dim gray=metadata). Git actions are prefixed with "Git:" in the action palette and only shown when actionable. Stage/unstage via `GitActionRunner` (async, follows `GitStatusLoader` pattern); `GitUtils.Stage`/`Unstage`/`StageAll`/`UnstageAll`/`Commit`/`Push`/`PushForceWithLease`/`Pull`/`PullRebase`/`Fetch` execute the corresponding git commands; network operations use 30s timeout. `GetAheadBehind` queries `git rev-list --count --left-right @{upstream}...HEAD`; result displayed in status bar as `ŌåæN ŌåōM` after branch name; multi-select supported via `_markedPaths`; status auto-refreshes after each action via `GitActionCompleteEvent`. Properties overlay (`i` key) shows a "Git status" row with colored text matching PaneRenderer colors; `PropertiesOverlay.Render` takes an optional `GitFileStatus?` parameter; `FormatGitStatus` is `internal static` for testability.
- **File finder** ŌĆö `Ctrl+F` recursive file search via `ScanFilesForFinder` (internal static, testable). Uses manual stack-based directory walk (not `RecurseSubdirectories`) so directories can be skipped before descending. `.git` directories are always excluded (even with `showHidden=true`). Dot-prefixed directories and hidden/system directories are skipped based on visibility settings. Depth limit 8, entry cap 10,000.
- **Cloud file placeholders** ŌĆö OneDrive/Dropbox/cloud-synced files that aren't fully downloaded locally are detected via Windows Cloud Files API attributes (`FILE_ATTRIBUTE_RECALL_ON_DATA_ACCESS` 0x00400000, `FILE_ATTRIBUTE_RECALL_ON_OPEN` 0x00004000). Cast `FileInfo.Attributes` to `int` and check bits ŌĆö no P/Invoke needed. Cloud placeholders keep their original file-type icon in the first column; a cloud icon (nf-md-cloud_outline U+F0163) is shown in the status column (same column as git status icons). Dim gray styling (140, 140, 160). Properties dialog shows the extension-based type label (e.g. "PDF", "C#") or "File"/"Directory" and adds "Cloud file" to the Attributes field. Metadata is shown via `FileMetadataProvider` (filename, size, modified date, "Cloud: not downloaded") ŌĆö `MetadataProviderRegistry` returns only `FileMetadataProvider` for cloud placeholders. Visual preview is blocked at the registry level (`PreviewProviderRegistry` returns empty for `IsCloudPlaceholder`); the preview pane renders "[cloud file ŌĆō not downloaded]" instead of attempting to read the file. Detection in `DirectoryContents.CheckIsCloudPlaceholder()`, stored on `FileSystemEntry.IsCloudPlaceholder`. Windows-only (always `false` on Unix).
- **Preview provider system** ŌĆö Two separate interfaces in `Wade.Preview`: `IPreviewProvider` for visual previews (text/image) and `IMetadataProvider` for structured metadata extraction. `IPreviewProvider` declares `CanPreview()` and returns a `PreviewResult` carrying text (`StyledLine[]`), image (Sixel data), or both. `IMetadataProvider` declares `CanProvideMetadata()` and returns `MetadataResult` with structured `MetadataSection[]`/`MetadataEntry[]`. `PreviewContext` carries pane dimensions, config flags (`ZipPreviewEnabled`, `ImagePreviewsEnabled`, `ArchiveMetadataEnabled`), git status, `IsCloudPlaceholder`, and `IsBrokenSymlink`. `MetadataProviderRegistry` short-circuits for `IsBrokenSymlink`; for cloud placeholders, only `FileMetadataProvider` is returned (reads filesystem attributes only, no file I/O ŌĆö shows filename, size, modified date, and "Cloud: not downloaded"); skips `ArchiveMetadataProvider` when `context.ArchiveMetadataEnabled` is false. `PreviewProviderRegistry` short-circuits for both `IsBrokenSymlink` and `IsCloudPlaceholder`. When `FilePreviewsEnabled` is false, App.cs returns an empty provider list instead of querying the registry. Preview providers in `PreviewProviders.cs` (registry order): `ImagePreviewProvider`, `PdfPreviewProvider`, `GlowMarkdownPreviewProvider`, `ZipContentsPreviewProvider`, `TextPreviewProvider` (non-binary only via `FilePreview.IsBinary`), `DiffPreviewProvider`, `NonePreviewProvider` (always applicable, returns placeholder), `HexPreviewProvider` (always applicable, explicit opt-in). Binary files default to None (not Hex) because None precedes Hex in registry order. Metadata providers (registry order): `FileMetadataProvider` (all files and directories ŌĆö filename as section header, size, modified date, git status if applicable; always first), `ImageMetadataProvider` (`.png`/`.jpg`/`.jpeg`/`.gif`/`.bmp`/`.webp`/`.tga`/`.tiff`/`.pbm` ŌĆö resolution, format, color depth, frame count via `Image.Identify()`; EXIF section with camera, date taken, exposure, focal length, GPS, software when present), `ExecutableMetadataProvider` (`.exe`/`.dll` ŌĆö PE headers via `PEReader`, .NET assembly metadata via `MetadataReader`, `FileVersionInfo` on Windows; NativeAOT-safe), `OfficeMetadataProvider` (`.docx`/`.xlsx`/`.pptx`/`.dotx`/`.xltx`/`.potx` ŌĆö OPC metadata from `docProps/core.xml` and `docProps/app.xml`), `MediaMetadataProvider` (audio/video ŌĆö shells out to `ffprobe`/`mediainfo`; `ParseFfprobeJson`/`ParseMediainfoJson` are `internal static` for testability), `NuGetMetadataProvider` (`.nupkg`/`.snupkg` ŌĆö `.nuspec` XML via `System.Xml.Linq` with local-name matching), `ArchiveMetadataProvider` (all zip-based archives ŌĆö file count, total/compressed size, compression ratio via `System.IO.Compression.ZipFile`; applies to all `ZipPreview.IsZipFile()` extensions; gated by `ArchiveMetadataEnabled` config), `PdfMetadataProvider` (`.pdf` ŌĆö shells out to `pdfinfo`). `MetadataRenderer` (`Wade.UI`) converts structured `MetadataSection[]` to `StyledLine[]` with colored label/value styling. `MetadataProviderRegistry.GetApplicableProviders()` returns a `List<IMetadataProvider>` of **all** applicable providers for a file (not just the first match); their sections are combined in order before injecting a single `MetadataReadyEvent`. `PreviewLoader.BeginLoad(path, metadataProviders, previewProvider, context)` loads metadata from all providers first (iterating in list order, merging sections, injecting `MetadataReadyEvent`), then loads the preview. App.cs stores `_applicableMetadataProviders` (list, no active index ŌĆö all sections are shown together). App.cs renders three display modes: metadata-only, preview-only, or metadata header above preview. `PropertiesOverlay.Render` accepts optional `MetadataSection[]` to show extracted metadata below file system properties. `PreviewProviderRegistry` provides an ordered list of applicable preview providers per file. Action palette supports submenus via a stack-based `ActionMenuLevel` model (`Wade.UI`). Preview providers are grouped under a "Change preview" submenu when multiple providers are available; also accessible directly via `p` keybinding. Combined text+image previews are supported via `CombinedPreviewReadyEvent`. To add a new metadata format: implement `IMetadataProvider` and add to `MetadataProviderRegistry.s_providers`. To add a new visual preview: implement `IPreviewProvider` and add to `PreviewProviderRegistry.s_providers`.
- **Config dialog** ŌĆö data-driven model in `Wade.UI`: `ConfigItem` (pure data: label, indent, format/toggle/cycle delegates, enabled predicate) and `ConfigDialogState` (owns mutable dialog state, builds items, handles navigation/toggle/cycle, copies to/from `WadeConfig` via `FromConfig`/`ApplyTo`). App.cs creates `ConfigDialogState` on dialog open, delegates input to its methods (`MoveUp`/`MoveDown`/`ToggleSelected`/`CycleNextSelected`/`CyclePrevSelected`), and renders from `Items`/`SelectedIndex`. No hardcoded index math ŌĆö items are built once with platform-conditional insertion (Windows-only "Show System Files"). Hierarchy: "Show Right Pane" gates "Show File Details" (with archive/PDF/media detail sub-items gated by `FileMetadata`) and "Show File Previews" (with image/PDF/archive/markdown preview sub-items gated by `FilePreviews`). Disabled items are skipped during navigation. `ConfigDialogStateTests` covers state, navigation, toggles, enabled predicates, and round-tripping.
- **Avoid Ctrl+Shift+&lt;key&gt; keybindings** ŌĆö Windows Terminal reserves many `Ctrl+Shift+` combinations (`C`, `V`, `P`, `W`, `Tab`, etc.) and `Ctrl+Tab`. Do not assign these as wade keybindings; they will be intercepted by the terminal and never reach the app.
- Uses `UseArtifactsOutput` ŌĆö build output goes to `artifacts/`, not `bin/`/`obj/`
